# DevTime — Система учёта рабочего времени

> Демо-версия на PostgreSQL. Все данные хранятся в БД и не сбрасываются между сессиями.

---

## 1. Архитектура и файловая структура

```
/
├── index.php        — Точка входа. Роутер по роли.
├── login.php        — Аутентификация.
├── logout.php       — Уничтожение сессии.
├── data.php         — Ядро: подключение к БД, все запросы, хелперы.
├── actions.php      — Обработчик POST-действий над сменами (start/stop/break/resume).
├── dashboard.php    — Дашборд. Панель управления + таблица учёта времени.
├── team.php         — Страница команд.
├── reports.php      — Экспорт отчёта в CSV/Excel.
├── admin.php        — Панель администратора. Полный CRUD пользователей и команд.
└── schema.sql       — Схема БД + дефолтные данные. Выполнить один раз в DBeaver.
```

## 2. Запуск

### Требования
- PHP >= 8.0 с расширениями `pdo` и `pdo_pgsql`
- PostgreSQL >= 13

### Включение расширений PHP

Найди `php.ini` и раскомментируй обе строки:

```ini
extension=pdo_pgsql
extension=pgsql
```

Перезапусти сервер после изменений.

### Настройка подключения к БД

В файле `data.php` в функции `getDB()`:

```php
$dsn = 'pgsql:host=127.0.0.1;port=5432;dbname=ИМЯ_БАЗЫ';
$pdo = new PDO($dsn, 'ЮЗЕР', 'ПАРОЛЬ', [...]);
```

### Создание базы данных (в DBeaver или psql)

```sql
CREATE DATABASE devtime OWNER postgres;
ALTER USER postgres WITH PASSWORD 'твой_пароль';
```

### Инициализация схемы

Открой `schema.sql` в DBeaver, выполни целиком — создаст все таблицы, роли, команды и дефолтных пользователей.

### Запуск сервера

```bash
cd /путь/к/проекту
php -S localhost:8000
```

Открой в браузере: `http://localhost:8000`

## 4. Схема базы данных

| Таблица | Назначение |
|---|---|
| `roles` | Справочник ролей: admin, teamlead, employee |
| `users` | Пользователи |
| `user_roles` | Связь пользователей и ролей |
| `teams` | Команды |
| `team_members` | Участники команд |
| `work_logs` | Журнал рабочего времени (интервальный подход) |
| `daily_reports` | Итоги дня: суммарные минуты, переработка |
| `overtime_requests` | Заявки на переработку (pending / approved / rejected) |

### Логика work_logs

Каждая строка — один непрерывный отрезок работы или перерыва:

| Действие | Что происходит в БД |
|---|---|
| Нажал «Начать» | `INSERT` с `started_at = NOW()`, `ended_at = NULL` |
| Нажал «Пауза» | `UPDATE ended_at = NOW()` + новый `INSERT` с `type = 'break'` |
| Нажал «Продолжить» | `UPDATE ended_at = NOW()` на перерыве + новый `INSERT` с `type = 'work'` |
| Нажал «Стоп» | `UPDATE ended_at = NOW()` + пересчёт `daily_reports` |

Переработка считается автоматически при «Стоп»: если сумма рабочих минут за день превышает 480 (8 часов), остаток записывается в `overtime_requests` со статусом `pending`.

## 5. Поток аутентификации

```
GET /index.php
    └─► Сессия есть?
            ├─ НЕТ  ──► login.php
            │               └─► POST → password_verify() против password_hash в БД
            │                         ├─ role=admin ──► admin.php
            │                         └─ иначе      ──► dashboard.php
            ├─ admin    ──► admin.php
            └─ иначе    ──► dashboard.php
```

## 6. Логика видимости данных по ролям

### Dashboard
| Что | Тим Лид | Сотрудник |
|---|---|---|
| Карточки управления | Все сотрудники | Только своя |
| Таблица учёта времени | Все сотрудники | Только своя строка |
| Итоговые карточки | Агрегат по всем | Только свои данные |

### Team
| Что | Тим Лид | Сотрудник |
|---|---|---|
| Видит команды | Все где состоит | Команды где состоит |
| CRUD команд | Да | Нет |

### Reports
Оба могут скачать CSV. Тим Лид получает все записи, сотрудник — только свои.
