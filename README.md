# DevTime — Система учёта рабочего времени

> Приложение на PHP + PostgreSQL для учёта рабочего времени IT-команды.

---

## 1. Файловая структура

```
/
├── index.php        — Точка входа. Роутер по роли.
├── login.php        — Аутентификация.
├── logout.php       — Уничтожение сессии.
├── data.php         — Ядро: настройки БД, все запросы, хелперы.
├── actions.php      — Обработчик смен (start / stop / break / resume).
├── dashboard.php    — Дашборд. Панель управления + таблица учёта времени.
├── team.php         — Страница команд.
├── reports.php      — Экспорт отчёта в CSV.
├── admin.php        — Панель администратора. CRUD пользователей и команд.
├── initbase.php     — Автоматически создаёт базу и схему при первом запуске.
└── schema.sql       — Схема БД для ручного выполнения в DBeaver (опционально).
```

---

## 2. Быстрый старт

### Требования
- PHP >= 8.0
- PostgreSQL >= 13

### Шаг 1 — Включи расширения PHP

Найди `php.ini` (путь покажет `<?php phpinfo();`) и раскомментируй:

```ini
extension=pdo_pgsql
extension=pgsql
```

Перезапусти сервер после изменений.

### Шаг 2 — Настрой подключение к БД

В файле `data.php` в самом верху:

```php
define('DB_HOST', '127.0.0.1');
define('DB_PORT', '5432');
define('DB_NAME', 'devtime');      // имя базы
define('DB_USER', 'postgres');     // пользователь
define('DB_PASS', 'твой_пароль'); // пароль
```

### Шаг 3 — Запусти сервер

```bash
cd /путь/к/проекту
php -S localhost:8000
```

### Шаг 4 — Инициализируй базу

Открой в браузере `http://localhost:8000` — если база не существует, произойдёт автоматический редирект на `initbase.php`, который создаст базу, схему и всех дефолтных пользователей. После этого нажми «Перейти к входу».

> Повторный запуск `initbase.php` безопасен — если таблицы уже есть, ничего не тронет.

## 4. Схема базы данных

| Таблица | Назначение |
|---|---|
| `roles` | Справочник ролей: admin, teamlead, employee |
| `users` | Пользователи |
| `user_roles` | Связь пользователей и ролей |
| `teams` | Команды |
| `team_members` | Участники команд |
| `work_logs` | Журнал рабочего времени (интервальный подход) |
| `daily_reports` | Итоги дня: суммарные минуты, переработка |
| `overtime_requests` | Заявки на переработку (pending / approved / rejected) |

### Логика work_logs

Каждая строка — один непрерывный отрезок работы или перерыва:

| Действие | Что происходит в БД |
|---|---|
| «Начать» | INSERT started_at = NOW(), ended_at = NULL |
| «Пауза» | UPDATE ended_at = NOW() + INSERT type = 'break' |
| «Продолжить» | UPDATE ended_at = NOW() на перерыве + INSERT type = 'work' |
| «Стоп» | UPDATE ended_at = NOW() + пересчёт daily_reports |

Переработка считается автоматически при «Стоп»: если сумма рабочих минут за день > 480 (8 часов), остаток записывается в `overtime_requests` со статусом `pending`.

## 6. Исправленные ошибки (code review)

### Критические

| Файл | Проблема | Исправление |
|---|---|---|
| `team.php` | CRUD команд работал через `$_SESSION` вместо БД (TODO-заглушки) | Переписан на реальные SQL-запросы с проверкой прав тим лида |
| `schema.sql` / `initbase.php` | В таблице `overtime_requests` отсутствовал `UNIQUE (user_id, report_date)`, из-за чего `ON CONFLICT` в `data.php` падал с ошибкой | Добавлен UNIQUE constraint в обоих файлах |
| `data.php` | `ON CONFLICT DO NOTHING` без указания колонок конфликта — невалидный синтаксис PostgreSQL | Исправлено на `ON CONFLICT (user_id, report_date) DO NOTHING` |

### Безопасность

| Файл | Проблема | Исправление |
|---|---|---|
| `logout.php` | `session_destroy()` без предварительной очистки данных сессии и удаления cookie — сессия могла не уничтожаться полностью | Добавлены `$_SESSION = []`, удаление cookie и проверка `session_status()` |
| `actions.php` | `employee_name` из POST передавался в URL редиректа без санитизации — потенциальный XSS | Добавлены `htmlspecialchars` + `strip_tags` перед передачей в заголовок |

### Логические ошибки

| Файл | Проблема | Исправление |
|---|---|---|
| `admin.php` | При отображении карточек команд тим лид искался как первый попавшийся пользователь с ролью `teamlead` из всех пользователей, а не из участников конкретной команды | Поиск переключён на `$members` (участники текущей команды) |
| `team.php` | Тернарный оператор в заголовке `'Мои команды' : 'Мои команды'` — обе ветки одинаковые | Исправлено на `'Мои команды' : 'Моя команда'` |

### Мелкие неточности

| Файл | Проблема | Исправление |
|---|---|---|
| `dashboard.php` | Живые часы обновлялись каждые 10 секунд, хотя отображаются только HH:MM | Интервал изменён на 60 секунд |
